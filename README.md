# MLOps Lab 3 — DVC + MLflow on Heart Disease Dataset

## Dataset
**UCI Heart Disease (Cleveland)** — binary classification: predict presence/absence of heart disease.  
Loaded directly from the UCI ML Repository; no manual download needed.

## Project Structure
```
├── preprocess.py          # Stage 1 – load, clean, split → data/train.csv + data/test.csv
├── train.py               # Stage 2 – Logistic Regression baseline, manual MLflow logging
├── tune.py                # Stage 3 – RandomForest grid search with nested MLflow runs
├── dvc.yaml               # DVC pipeline definition (3 stages)
├── .env.example           # Template for DagsHub credentials
└── data/                  # Generated by DVC (git-ignored)
```

## Setup

### 1. Clone & create virtual environment
```bash
git clone https://github.com/<your-username>/mlops_lab3_v3.git
cd mlops_lab3_v3
python -m venv .venv
# Windows PowerShell:
.venv\Scripts\Activate.ps1
# macOS/Linux:
source .venv/bin/activate
```

### 2. Install dependencies
```bash
pip install dvc mlflow scikit-learn pandas python-dotenv joblib
```

### 3. Configure DagsHub credentials
```bash
cp .env.example .env
# Edit .env and fill in your real MLFLOW_TRACKING_URI, USERNAME, PASSWORD
```
Get your values from: **DagsHub repo → Remote → MLflow**.

### 4. Run the full pipeline
```bash
dvc repro
```
DVC resolves the dependency order automatically: `preprocess → train → tune`.

To run a single stage:
```bash
dvc repro tune   # also reruns preprocess & train if needed
```

### 5. View results
Open your DagsHub repository → **Experiments** tab to see all 3 runs side-by-side.
